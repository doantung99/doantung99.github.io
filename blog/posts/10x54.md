## Summary
`ducktricks.bin` is an ESP32-S3 Arduino image hiding "VaultPass v1.3", a 16-byte serial crackme that decrypts a 38-byte flag with `flag[i] = i ^ cipher[i] ^ key[i%16]` where `key` is an LCG keystream. Because the LCG state is fully recoverable from a few known output bytes, I never had to pass the actual checks — the `V1T{` crib alone pins the keystream and unrolls the whole flag.

## Solution
I clocked this as an ESP32 firmware crackme the moment I saw a 3 MB blob with no useful strings, so I set the direction: triage the image, find the obfuscation helper, then locate the flag-decrypt math. I handed the binary to the model and had it run `esptool image_info` first — it confirmed only the first ~286 KB is real (6 segments + footer) and that the two appended regions at `0x46000`/`0x88000` are decoy Xtensa blobs. I told it to ignore those and focus on `.rodata`.

The model then found a 60-byte binary blob at the very start of `.rodata` (`0x3c030120`) and disassembled the helper at `0x1a4c` that consumes it — a plain `xor a11, a11, 0x42` before `Serial.write`. Batch-decoding with `^0x42` gave `DENIED`, `GRANTED`, `KEY>`, `VaultPass v1.3`: a 16-byte serial check. Its first instinct was to reverse the full validation state machine (XOR checksum vs `0x57`, CRC `0xfdd0`/`0x9456`, the LCG). I corrected course — we don't need the password, we need the keystream — and pointed it at the GRANTED decrypt loop at `0x1c48`.

From there the model reversed the `cipher` assembly (the 38-byte blob `d[0x146:0x16c]` with its three thirds reversed) and the LCG (`x = x*0x6c62272e + 0x07354a6b`, `key[i] = x>>24`). I supplied the key judgment call from the previous hardware challenge: the prompts use **uppercase** `V1T{`. That crib gives `key[0..3]`, which uniquely pins the 32-bit LCG state — I had the model brute-force the low 24 bits and verify the full keystream decrypted to clean ASCII. It did.

```python
# Run on the extracted .rodata bytes; d = the 38-byte blob region of the ESP image.
# d[0x146:0x16c] is the cipher source inside seg0 DROM (.rodata).
M, C, MASK = 0x6c62272e, 0x07354a6b, 0xffffffff

# 38-byte blob: thirds reversed (last third first)
cipher = d[0x160:0x16c] + d[0x153:0x160] + d[0x146:0x153]

# Known-plaintext crib: flag starts with uppercase V1T{
k = [i ^ cipher[i] ^ ord(c) for i, c in enumerate("V1T{")]   # key[0..3]

for lo in range(1 << 24):
    x1 = (k[0] << 24) | lo
    x2 = (x1 * M + C) & MASK
    if (x2 >> 24) != k[1]: continue
    x3 = (x2 * M + C) & MASK
    if (x3 >> 24) != k[2]: continue
    x4 = (x3 * M + C) & MASK
    if (x4 >> 24) != k[3]: continue
    # state pinned: regenerate all 16 key bytes and decrypt 38 bytes
    ks, x = [(x1 >> 24) & 0xff], x1
    for _ in range(15):
        x = (x * M + C) & MASK
        ks.append((x >> 24) & 0xff)
    flag = bytes(i ^ cipher[i] ^ ks[i % 16] for i in range(38))
    print(flag.decode())     # V1T{LF5R_1s_tr4sh_wH0_n33ds_p4sS_lMa0}
    break
```

The uppercase crib yields a single LCG state whose keystream (`7ab01cdbe475f2174a298fd5da73b7ad`) decrypts all 38 bytes to clean ASCII — no password-cracking required.
